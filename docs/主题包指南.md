# 使用说明

## 1. 选择主题包

进入 `Codex主题包/` 或 `WorkBuddy主题包/`，下载对应目录中的 `.codedrobe-theme`。该文件是 UTF-8 JSON，不是 ZIP，不需要解压。

## 2. 安装运行时

```bash
npm install --global @codedrobe/core@0.3.0
codedrobe --version
codedrobe apps --json
```

Node.js 必须为 22.4 或更高版本。应用路径和端口以 `codedrobe apps --json`、`codedrobe detect` 的实际输出为准。

## 3. 安装前检查

```bash
codedrobe theme inspect /absolute/theme.codedrobe-theme
codedrobe probe --app codex --theme /absolute/theme.codedrobe-theme --timeout-ms 5000
```

WorkBuddy 使用 `--app workbuddy`。`probe` 只检查 DOM 前提，不代表视觉验收完成。

## 4. 应用与验证

```bash
codedrobe apply --app codex --theme /absolute/theme.codedrobe-theme
codedrobe verify --app codex --theme /absolute/theme.codedrobe-theme --screenshot /absolute/result.png
```

必须分别检查首页和普通会话：导航、输入框、发送按钮、菜单、代码块、滚动和焦点不能被主题破坏。未经明确授权，不要增加 `--restart-existing`。

## 5. 恢复

```bash
codedrobe restore --app codex
```

WorkBuddy 使用 `codedrobe restore --app workbuddy`。不要把删除应用数据、改写 `app.asar` 或重装客户端当作常规恢复方案。

# 主题制作指南

## 事实源

每套主题只在 `shared-theme-sources/<theme-id>/` 维护一份设计事实源：

- `theme-source.json.metadata`：名称、版本、原创声明和兼容状态；
- `theme-source.json.tokens`：颜色、圆角、阴影与动效参数；
- `theme-source.json.assets`：四层可审查原创 SVG 字符串与一张徽记 PNG Base64；
- `theme-source.json.css`：Codex 与 WorkBuddy 双端独立 CSS。

## 制作步骤

1. 使用安全英文 ID，不包含第三方作品、角色或商标名称；
2. 绘制原创 `hero`、`texture`、`emblem`、`ambient` SVG；
3. 将 SVG 转为 PNG，包内只嵌入 PNG/JPEG/WebP/GIF；
4. Codex CSS 全部限定在 `html.codedrobe-host-codex`；
5. WorkBuddy CSS 全部限定在 `html.codedrobe-host-workbuddy`；
6. 装饰层设置 `pointer-events: none`；
7. 不使用远程 `@import`、远程 `url(...)`、JavaScript 或事件处理器；
8. 执行 `npm run check`；
9. 在真实客户端执行 probe、apply、verify、截图和 restore 后，才能把实机状态从 `NOT_RUN` 改为 `PASS`。

## CodeDrobe 包结构

`.codedrobe-theme` 为 UTF-8 JSON，核心字段为：

```json
{
  "format": "codedrobe-theme",
  "schemaVersion": 1,
  "theme": {"id": "example-codex", "displayName": "Example", "version": "1.0.0"},
  "targets": {"codex": {"css": "html.codedrobe-host-codex body {}"}},
  "assets": {
    "images": {
      "hero": {"filename": "hero.png", "mimeType": "image/png", "base64": "..."}
    }
  }
}
```

单个公开包只保留一个 target，便于下载、排障和兼容性记录。
